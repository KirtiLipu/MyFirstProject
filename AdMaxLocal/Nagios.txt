Exit Codes:
==============
Nagios understands the following exit codes:

    0 - Service is OK.
    1 - Service has a WARNING.
    2 - Service is in a CRITICAL status.
    3 - Service status is UNKNOWN.

=====================================
there are plugins for:

    HTTP, POP3, IMAP, FTP, SSH, DHCP
    utilization of CPU, hard disk drive and main memory, active users
    Operating systems like Windows, Linux/Unix distributions
    Routers, switches
    Log files

main Nagios config file.
vi /etc/nagios/nagios.cfg

Sample configuration files are installed in the /usr/local/nagios/etc/ directory

Every time you modify your configuration files, you should run a sanity check on them.
verify your configuration, run Nagios :

/usr/bin/nagios -v /etc/nagios/nagios.cfg
nagios reload
sudo /etc/init.d/nagios reload

What Are Plugins?
Plugins are compiled executables or scripts (Perl scripts, shell scripts, etc.) that can be run from a command line to check the status or a host or service.

What Plugins Are Available?
There are plugins currently available to monitor many different kinds of devices and services, including:
HTTP, POP3, IMAP, FTP, SSH, DHCP
CPU Load, Disk Usage, Memory Usage, Current Users
Unix/Linux, Windows, and Netware Servers
Routers and Switches
etc.
